Security & Resilience Architecture

Verified Compliance: NIST 2.0 & DORA (Digital Operational Resilience Act)

1. Identity & Access Management (IAM)

SafeTrade Insider utilizes a Zero-Trust Architecture. Our automated backend (The Brain) communicates with the Terminal via scoped, encrypted tokens.

  • Personal Access Tokens (PAT): We utilize GitHub Fine-Grained PATs with "Least Privilege" scopes. This ensures the automation can only update the signals.json file and has no access to sensitive account settings.
  • Machine-to-Machine (M2M) Security: All handshakes between Make.com and GitHub are encrypted using TLS 1.3.
  • Credential Rotation: Our API keys for FRED, CoinGecko, and GitHub are rotated every 90 days in accordance with NIST 2026 guidelines.

2. Data Integrity & "The Brain" Workflow

To prevent market manipulation or data tampering, we employ strict validation protocols:

  • Source Verification: Data is pulled only from verified institutional endpoints (St. Louis Fed, CoinGecko Enterprise).
  • Algorithmic Validation: Before a "CRITICAL" signal is published, the logic must pass a 3-point correlation check between the US Dollar Index (DXY) and BTC Liquidity.
  • Static Delivery: By using GitHub Pages as our delivery layer, we eliminate database vulnerabilities. The website is "Read-Only" for the public, making it immune to SQL injection attacks.
